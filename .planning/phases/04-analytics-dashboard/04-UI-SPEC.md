## Spacing Scale

Declared values (multiples of 4 only). Matches Tailwind 4-point scale already in use project-wide.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon-to-label gaps, chip internal padding |
| sm | 8px (`gap-2`, `p-2`) | Compact row padding, badge gaps |
| md | 16px (`gap-4`, `p-4`) | Default element spacing, Card `padding="sm"` (p-4) |
| lg | 24px (`gap-6`, `p-6`) | Page section gaps, Card `padding="md"` (p-6) |
| xl | 32px (`gap-8`, `p-8`) | Layout gaps between major page sections, Card `padding="lg"` (p-8) |
| 2xl | 48px | Major section breaks (empty state vertical padding: `py-12`) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Stat card icon container: `w-10 h-10` (40px square) — matches existing DashboardPage pattern
- Touch targets for filter chips: minimum `h-8` (32px) — compact chip pattern
- Table cell padding: `px-4 py-3` — matches CampaignsPage table pattern exactly

Source: `src/components/ui/Card.tsx` padding variants; `src/pages/dashboard/DashboardPage.tsx`; `src/pages/campaigns/CampaignsPage.tsx` table cell pattern.

| Role | Tailwind Value | Hex Approx | Usage |
|------|---------------|------------|-------|
| Dominant (60%) | `bg-gray-950` | #030712 | Page background (`AppLayout`) |
| Secondary (30%) | `bg-gray-900` | #111827 | Card backgrounds, table headers |
| Accent (10%) | `bg-indigo-600` / `text-indigo-400` | #4f46e5 / #818cf8 | Primary CTA buttons, icon tint for "Total Campaigns" stat card only |
| Destructive | `bg-red-700` / `text-red-400` | #b91c1c / #f87171 | Delete actions only |

Accent reserved for:
- "View analytics" navigation link text in campaign table dropdown (ghost button, no fill)
- Primary CTA button if present (variant="primary" → `bg-indigo-600`)
- Spinner border accent (`border-t-indigo-500`)
- Active filter chip background (`bg-indigo-600/20 text-indigo-400 border-indigo-600/30`)
- Stat card icon container for "Total Campaigns" card — `text-indigo-400` (matches DashboardPage existing pattern)

Stat card icon color assignments (matching established DashboardPage pattern):
- Total Contacts → `text-blue-400`
- Campaigns Sent → `text-indigo-400`
- Avg Open Rate → `text-green-400`
- Avg Click Rate → `text-purple-400`

Event type color assignments (for timeline icon backgrounds):
- `opened` → `text-green-400` (success-adjacent)

- `bounced` → `text-red-400` (danger)

## Interaction Contracts

### Filter Chips (Event Timeline)

- Chip variants: All / Opened / Clicked / Bounced / Unsubscribed
- Inactive state: `Button variant="ghost"` — `text-gray-400 hover:text-gray-200 hover:bg-gray-800`
- Active state: custom class override — `bg-indigo-600/20 text-indigo-400 border border-indigo-600/30 rounded-lg`
- Only one chip active at a time; clicking active chip resets to "All"
- Filter triggers server-side re-fetch (not client-side array filter) — hook re-queries with `event_type` param

### Recipient Engagement Tabs

- Tab strip: All / Opened / Clicked / Bounced / Unsubscribed
- Each tab shows count badge: `(N)` computed client-side from full first-page load
- Active tab: `text-indigo-400 border-b-2 border-indigo-400`
- Inactive tab: `text-gray-400 hover:text-gray-200`
- Tab selection triggers server-side re-fetch with `status` filter

### Expandable Table Row (Recipient Engagement)

- Default row: email address + status badge + expand chevron (`ChevronDown` / `ChevronUp`)
- Click anywhere on row to expand/collapse — uses `data-no-list-click` pattern on expand icon only
- Expanded detail row: spans full table width with `colSpan`
- Detail layout: timestamp grid (2 columns, 3 rows) showing `sent_at`, `opened_at`, `clicked_at`, `bounced_at`, `unsubscribed_at` + device/client info
- Null timestamps display as `—`
- Only one row expanded at a time (`expandedId` state)
- Expanded section background: `bg-gray-800/30` to distinguish from main row

### Pagination Controls

- Prev / Next buttons: `Button variant="ghost" size="sm"`
- Disabled state when on first/last page: `disabled` prop triggers `opacity-50 cursor-not-allowed` via Button component
- Page indicator: `text-sm text-gray-400` — "Page N of M"
- Page size: 50 rows (matches `useContacts` pattern)

### Dashboard Recent Campaigns Row Click

- Full table row is clickable: `cursor-pointer hover:bg-gray-800/50` — navigates to `/campaigns/:id/analytics`
- Pattern: identical to CampaignsPage row click using `data-no-list-click` guard

### Loading States

- Per-section Spinner (not full-page): each section renders `Spinner size="lg"` centered when `loading === true`
- Spinner centered: `flex items-center justify-center py-12`
- Dashboard stat cards: show `—` (em dash) while loading — matches existing DashboardPage placeholder pattern
- No skeleton screens — Spinner matches established project pattern

### Hover Tooltip for Absolute Timestamp

- Relative timestamp (`2 hours ago`) renders on screen
- `title` attribute on the element contains absolute ISO formatted time: `new Date(ts).toLocaleString()`
- No custom tooltip component — browser native title tooltip is sufficient for MVP

## Accessibility Notes

- All interactive table rows include `cursor-pointer` and are keyboard-focusable via `tabIndex={0}` with `onKeyDown` Enter/Space handlers
- Filter chips and tab strips use `<button>` elements (not `<div>`) for native keyboard access
- Spinner has `role="status"` and `aria-label="Loading"` — already implemented in existing Spinner component
- Icon-only buttons (expand chevron) include `aria-label` describing the action
- Relative timestamps include `title` attribute with absolute ISO time for screen readers and power users
- URL column in link breakdown: full URL in `title` attribute; truncated via `truncate max-w-xs` CSS class
