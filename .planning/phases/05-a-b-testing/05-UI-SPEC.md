## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (e.g. `gap-1`, `px-1`) |
| sm | 8px | Compact element spacing (e.g. `gap-2`, `p-2`) |
| md | 16px | Default element spacing (e.g. `gap-4`, `p-4`) |
| lg | 24px | Section padding (e.g. `gap-6`, `p-6`) |
| xl | 32px | Layout gaps (e.g. `gap-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets for icon-only buttons: 32px × 32px (`w-8 h-8`) — matches existing `Button ghost sm` pattern in CampaignsPage
- Stat card icon containers: 40px × 40px (`w-10 h-10`) — matches existing StatCard component
- Test group size slider/input thumb: minimum 44px touch target height on mobile

Source: extracted from `CampaignsPage.tsx`, `AppLayout.tsx` (`p-4 lg:p-6`), `StatCard.tsx`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label | 12px (`text-xs`) | 400 regular / 600 semibold for table headers | 1.5 |
| Heading | 20px (`text-xl`) | 600 (semibold) | 1.2 |
| Display (stat value) | 24px (`text-2xl`) | 600 (semibold) | 1.2 |

Notes:
- Page heading: `text-xl font-semibold text-gray-100` — matches `CampaignsPage` `<h1>`
- Table column headers: `text-xs font-semibold text-gray-400 uppercase tracking-wide` — matches existing table pattern
- Stat card values: `text-2xl font-semibold text-gray-100` — reuse StatCard as-is from Phase 4
- Section sub-labels: `text-xs text-gray-400`
- Exactly 2 weights used: regular (400) and semibold (600)

Source: `CampaignsPage.tsx`, `StatCard.tsx`, `Card.tsx`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `gray-950` (`#030712`) | Page background — `bg-gray-950` on AppLayout root |
| Secondary (30%) | `gray-900` (`#111827`) | Cards (`bg-gray-900`), sidebar, nav, table backgrounds |
| Accent (10%) | `indigo-600` (`#4f46e5`) | Primary CTA buttons only |
| Destructive | `red-700` (`#b91c1c`) | "Send winner" danger action button only |

Accent reserved for: "New A/B Test" button (primary CTA on CampaignsPage), "Send variants" submit button on A/B builder page, "Confirm send winner" button inside confirmation prompt only. Not used on tabs, badges, stat cards, or links.

Additional semantic colors (not accent, not destructive):
- `indigo-400 / indigo-900/50`: A/B test badge on campaign list row (info variant — matches existing `Badge variant='info'`)
- `green-400 / green-900/50`: "Winner sent" success badge (success variant — matches existing `Badge variant='success'`)
- `gray-700 text-gray-200`: Neutral/default badge (used for "A/B Test" type label when test is in progress)
- `yellow-400 / yellow-900/50`: "Sending" status on variant rows (warning variant)

Source: `Button.tsx` (`indigo-600` primary, `red-700` danger), `Badge.tsx` (semantic variants), `AppLayout.tsx`, `Card.tsx`.

## Layout Patterns

### A/B Test Builder Page (`/campaigns/ab-test/new`, `/campaigns/:id/ab-test/edit`)

```
Page header row
  "A/B Test" breadcrumb label  |  [Save Draft]  [Send Test Variants]

Shared Settings section (Card)
  From name  |  From email  |  Reply-to
  Contact list selector
  Test group size input + derived breakdown: "Variant A: 20% · Variant B: 20% · Hold-back: 60%"

Variant Tab Strip
  [Variant A]  [Variant B]  ← tabs, only one active at a time

Variant Editor panel (per active tab)
  Subject line input (full width)
  TipTap rich text editor (identical config to CampaignBuilderPage)
```

### A/B Test Results Page (`/campaigns/:id/ab-results`)

```
Page header row
  Back link  |  A/B Test name  |  Status badge

Split breakdown summary bar
  "Variant A: 20% (N contacts) · Variant B: 20% (N contacts) · Hold-back: 60% (N contacts)"

Side-by-side stat cards grid (2 columns)
  Variant A column header  |  Variant B column header
  StatCard: Open Rate       |  StatCard: Open Rate
  StatCard: Click Rate      |  StatCard: Click Rate
  StatCard: Sent            |  StatCard: Sent
  StatCard: Opened          |  StatCard: Opened
  StatCard: Clicked         |  StatCard: Clicked

Send winner section (Card, only visible when both variants have been sent)
  Instruction text
  [Send Variant A to N contacts]  [Send Variant B to N contacts]
```

### CampaignsPage additions (existing page, additive only)

- "New A/B Test" button added to the header row alongside "New Campaign" (secondary variant)
- A/B test rows in the campaign list table show "A/B Test" type badge (Badge `variant='default'` with custom teal/cyan tint — use `bg-teal-900/50 text-teal-400` via className override)
- Clicking an A/B test row navigates to `/campaigns/:id/ab-test/edit`

## Interaction States

### Variant Tab Strip

| State | Visual |
|-------|--------|
| Inactive tab | `text-gray-400 border-b-2 border-transparent hover:text-gray-200` |
| Active tab | `text-gray-100 border-b-2 border-indigo-500` |
| Tab container | `border-b border-gray-800 flex gap-6 mb-6` |

### Split Percentage Input

| State | Visual |
|-------|--------|
| Default | Number input 8–90, default value 40 |
| Derived display | `text-sm text-gray-400` below input: "Variant A: X% · Variant B: X% · Hold-back: Y%" |
| Invalid (< 8 or > 90) | `text-sm text-red-400`: "Test group must be between 8% and 90%" |

### Send Variants Button (builder page)

| State | Visual |
|-------|--------|
| Default | `Button variant='primary' size='md'` — "Send test variants" |
| Loading | `Button loading={true}` — spinner replaces label |
| Disabled (variants incomplete) | `Button disabled` — muted opacity |

### Send Winner Flow

| State | Visual |
|-------|--------|
| Trigger | `Button variant='secondary'` per variant: "Send Variant A" / "Send Variant B" |
| Confirmation modal | `SendWinnerModal` — overlay with `bg-gray-900 border border-gray-800 rounded-lg p-6` |
| Confirm button inside modal | `Button variant='danger'` — "Send Variant A to N contacts" |
| Post-send | Replace send buttons with success state: `Badge variant='success'` "Winner sent" |
