## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), badge padding (`py-0.5`) |
| sm | 8px | Compact element spacing (`gap-2`), button icon gap |
| md | 16px | Default element spacing (`gap-4`), section inner padding |
| lg | 24px | Section gaps (`gap-6`), card padding default (`p-6`) |
| xl | 32px | Builder section gaps (`gap-8`) |
| 2xl | 48px | Major section breaks, empty state padding (`py-16`) |
| 3xl | 64px | Page-level spacing (unused in existing pages, available) |

Exceptions:
- Step number badge: 28px min-width circle (`w-7 h-7`) ‚Äî visual affordance, not a spacing token
- Nav icon touch target: 20px (`w-5 h-5`) within 40px (`px-3 py-2.5`) nav link hit area
- TipTap editor minimum height: 320px (`min-h-[320px]`) ‚Äî functional constraint, not a spacing token

Source: established from existing pages (`CampaignsPage.tsx`, `AbTestBuilderPage.tsx`, `CampaignBuilderPage.tsx`).

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body + Labels | 14px | 400 (regular) | 1.5 | `text-sm` |
| Section heading | 12px | 600 (semibold) | 1.2 | `text-xs font-semibold uppercase tracking-wide` |
| Page heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` |

Notes:
- Body text: `text-gray-100` for primary content, `text-gray-300` for secondary, `text-gray-400` for metadata/labels
- Section headings always uppercase with `tracking-wide` ‚Äî matches existing `Details`, `Shared Settings`, etc. sections
- TipTap editor body uses `text-sm leading-relaxed` (`leading-relaxed` = 1.625)
- Inline-editable sequence name input: `text-xl font-semibold` matching `CampaignBuilderPage` pattern
- Only two weights are in use: 400 (regular) for all body copy and labels, 600 (semibold) for headings and strong emphasis

Source: extracted from `CampaignBuilderPage.tsx`, `AbTestBuilderPage.tsx`, `AbTestResultsPage.tsx`.

## Page Layouts

### SequencesPage (`/sequences`) ‚Äî List Page

Layout mirrors `CampaignsPage.tsx` exactly:

```
flex flex-col gap-6
  ‚îú‚îÄ‚îÄ Page header row (flex items-center justify-between)
  ‚îÇ     ‚îú‚îÄ‚îÄ h1 "Sequences" (text-xl font-semibold text-gray-100)
  ‚îÇ     ‚îî‚îÄ‚îÄ Button primary "New Sequence" (Plus icon + label)
  ‚îî‚îÄ‚îÄ Card padding="sm" overflow-hidden p-0
        ‚îú‚îÄ‚îÄ [loading]  ‚Üí centered Spinner size="lg" py-16
        ‚îú‚îÄ‚îÄ [empty]    ‚Üí EmptyState (see Copywriting Contract)
        ‚îî‚îÄ‚îÄ [data]     ‚Üí <table> full-width
```

Table columns:
| Column | Content | Width |
|--------|---------|-------|
| Sequence | name (`text-sm font-semibold text-gray-100`) | auto |
| Status | `<Badge>` variant mapped from sequence status | auto |
| Contact list | list name or `‚Äî` (`text-sm text-gray-300`) | auto |
| Steps | step count + "steps" label (`text-sm text-gray-400`) | auto |
| Enrolled | enrollment count + "enrolled" label (`text-sm text-gray-400`) | auto |
| Actions | `MoreHorizontal` ghost button ‚Üí dropdown menu | 48px |

Table header: `text-xs font-semibold text-gray-400 uppercase tracking-wide bg-gray-900 px-4 py-3 text-left`
Table rows: `border-b border-gray-800 hover:bg-gray-800/50 cursor-pointer`
Row click ‚Üí navigate to `/sequences/:id/edit` (if draft/paused) or `/sequences/:id/results` (if active/archived)

Dropdown menu options (per sequence status):
- **Draft:** Edit, Delete
- **Active:** View Results, Pause, Archive
- **Paused:** View Results, Resume, Archive
- **Archived:** View Results (read-only; no delete if enrollments exist)

Delete confirmation: `window.confirm('Delete "{name}"? This cannot be undone.')` ‚Äî matches `CampaignsPage` pattern.
Archive confirmation: `window.confirm('Archive "{name}"? Active enrollments will stop receiving steps.')` ‚Äî inline confirm.

Step reorder UX: Up/Down arrow buttons (`ChevronUp`/`ChevronDown` from lucide-react, ghost size sm) placed in step header row, between step badge and remove button. First step hides ChevronUp; last step hides ChevronDown. Drag-and-drop is deferred.

Step validation: delay_days must be strictly increasing (step N delay_days > step N-1 delay_days). Inline error displayed beneath delay row: `text-sm text-red-400`.

"Start Sequence" button:
- Disabled if: sequence has 0 steps, any step is missing subject or has empty body, contact list not selected
- Requires confirmation modal (not `window.confirm`) ‚Äî see Interaction States below
- After start: navigate to `/sequences/:id/results`

"Save draft" button: always available, no validation required. Saves current state silently.

## Interaction States

### Start Sequence ‚Äî Confirmation Modal (`StartSequenceModal`)

Not `window.confirm`. Uses a custom modal matching `SendWinnerModal` pattern.

Modal content:
- Heading: "Start this sequence?"
- Body: "This will enroll {N} active contacts from {list name}. Emails will begin sending automatically. Contacts added after this point will not be enrolled."
- Actions: `[Keep Editing (secondary)]` `[Start Sequence (primary, loading state)]`

### Pause / Resume ‚Äî Inline confirmation

Pause: `window.confirm('Pause this sequence? Active contacts will stop receiving steps until you resume.')` ‚Äî acceptable because it is a recoverable action.
Resume: No confirmation required. Button label: "Resume" (primary sm).

### Delete ‚Äî `window.confirm` inline

Copy: `'Delete "{name}"? This cannot be undone.'`
Only available on draft sequences with no enrollments. Sequences with any enrollment record are archived, not deleted.

### Loading States

- List page: `<Spinner size="lg" />` centered in card, `py-16`
- Builder page: `<Spinner size="lg" />` centered `h-64` (edit mode only, while sequence loads)
- Results page: `<Spinner size="lg" />` centered `py-24`
- Save draft button: `loading` prop on Button (spinner inline)
- Start Sequence button: `loading` prop on Button

### Accessibility: Icon-only Button Labels

Every icon-only interactive element must declare an explicit `aria-label`:

| Element | Icon | `aria-label` |
|---------|------|-------------|
| Sequence row actions button | `MoreHorizontal` | `aria-label="Sequence actions"` |
| Step remove button | `X` | `aria-label="Remove step {N}"` (interpolated) |
| Step move-up button | `ChevronUp` | `aria-label="Move step up"` |
| Step move-down button | `ChevronDown` | `aria-label="Move step down"` |

### Error States

- Builder validation: inline `text-sm text-red-400` beneath each invalid field, `role="alert"`
- Delay ordering violation: "Step delays must be strictly increasing (each step's day must be greater than the previous step's day)."
- Network error (toast): uses existing `showToast(message, 'error')` pattern
- Results page load error: `text-sm text-red-400` inline ‚Äî "Failed to load sequence results. Try refreshing the page."
