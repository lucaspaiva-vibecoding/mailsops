  <action>
Create `supabase/functions/resend-webhook/index.ts`. This function has `verify_jwt = false` in config.toml (already created in Plan 01) because Resend's webhook calls carry no JWT — authentication is via Svix signature.

**Imports:**
```typescript
import { Webhook } from 'npm:svix'
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'
```

**Service-role client** (module-level):
```typescript
const supabase = createClient(
  Deno.env.get('SUPABASE_URL') ?? '',
  Deno.env.get('SUPABASE_SERVICE_ROLE_KEY') ?? ''
)
```

**Main handler — `Deno.serve(async (req) => { ... })`:**

1. **Only accept POST:**
   ```typescript
   if (req.method !== 'POST') {
     return new Response('Method not allowed', { status: 405 })
   }
   ```

2. **Read raw body FIRST (CRITICAL — per Research pitfall #2):**
   ```typescript
   const payload = await req.text()
   ```
   NEVER do `await req.json()` before verification — JSON re-serialization breaks the HMAC signature.

3. **Extract Svix headers:**
   ```typescript
   const svixHeaders = {
     'svix-id': req.headers.get('svix-id') ?? '',
     'svix-timestamp': req.headers.get('svix-timestamp') ?? '',
     'svix-signature': req.headers.get('svix-signature') ?? '',
   }
   ```

4. **Verify signature:**
   ```typescript
   const webhookSecret = Deno.env.get('RESEND_WEBHOOK_SECRET') ?? ''
   const wh = new Webhook(webhookSecret)
   let event: { type: string; created_at: string; data: { email_id: string; bounce?: { type: string } } }

   try {
     event = wh.verify(payload, svixHeaders) as typeof event
   } catch {
     return new Response('Invalid signature', { status: 400 })
   }
   ```
   Returning 400 (not 200) on invalid signature causes Resend to retry with the correct signature. Per D-06, signature verification protects the webhook endpoint.

5. **Map event type to status and counter field:**
   ```typescript
   const statusMap: Record<string, string> = {
     'email.delivered': 'delivered',
     'email.bounced': 'bounced',
     'email.complained': 'complained',
   }

   const counterMap: Record<string, string> = {
     'email.delivered': 'total_delivered',
     'email.bounced': 'total_bounced',
     'email.complained': 'total_unsubscribed',
   }

   const newStatus = statusMap[event.type]
   if (!newStatus) {
     // Unhandled event type — acknowledge to prevent retries
     return new Response('ok', { status: 200 })
   }
   ```

6. **Find recipient by resend_email_id:**
   ```typescript
   const { data: recipient } = await supabase
     .from('campaign_recipients')
     .select('id, campaign_id, contact_id, delivery_status')
     .eq('resend_email_id', event.data.email_id)
     .single()

   if (!recipient) {
     // No matching recipient — may be from a test send, acknowledge
     return new Response('ok', { status: 200 })
   }
   ```

7. **Update campaign_recipients delivery_status:**
   ```typescript
   const updatePayload: Record<string, string> = {
     delivery_status: newStatus,
   }

   if (event.type === 'email.delivered') {
     updatePayload.delivered_at = event.created_at
   } else if (event.type === 'email.bounced') {
     updatePayload.bounced_at = event.created_at
   }

   await supabase
     .from('campaign_recipients')
     .update(updatePayload)
     .eq('id', recipient.id)
   ```

8. **Update contact status for bounces:**
   ```typescript
   if (event.type === 'email.bounced') {
     await supabase
       .from('contacts')
       .update({
         status: 'bounced',
         bounce_type: event.data.bounce?.type === 'Permanent' ? 'hard' : 'soft',
         bounced_at: event.created_at,
       })
       .eq('id', recipient.contact_id)
   }

   if (event.type === 'email.complained') {
     await supabase
       .from('contacts')
       .update({ status: 'complained' })
       .eq('id', recipient.contact_id)
   }
   ```

9. **Increment campaign aggregate counter:**
   ```typescript
   const counterField = counterMap[event.type]
   if (counterField) {
     const { data: campaign } = await supabase
       .from('campaigns')
       .select(`id, ${counterField}`)
       .eq('id', recipient.campaign_id)
       .single()

     if (campaign) {
       await supabase
         .from('campaigns')
         .update({ [counterField]: (campaign as Record<string, number>)[counterField] + 1 })
         .eq('id', recipient.campaign_id)
     }
   }
   ```

10. **Return 200 OK:**
    ```typescript
    return new Response('ok', { status: 200 })
    ```

Wrap entire handler in try/catch. On error, return 500 with error message (Resend will retry).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Resend -> /resend-webhook | External webhook calls from Resend servers (no JWT, authenticated by Svix signature) |
| Webhook -> campaign_recipients | Updates delivery status based on trusted Resend events |
| Webhook -> contacts | Updates contact bounce/complaint status |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-14 | Spoofing | Webhook endpoint | mitigate | Svix HMAC-SHA256 signature verification on every request; invalid signatures return 400; `npm:svix` handles constant-time comparison and timestamp window check (5-minute window prevents replay) per D-06 |
| T-03-15 | Repudiation | Webhook replay | mitigate | Svix timestamp window check (built into `wh.verify`) prevents replay of old webhook payloads beyond 5-minute window |
| T-03-16 | Tampering | Webhook body | mitigate | Raw body read via `req.text()` before any parsing; Svix verifies exact bytes match the HMAC; JSON re-parsing would break verification |
| T-03-17 | Denial of Service | Webhook flood | accept | Supabase Edge Functions have built-in rate limiting; webhook operations are lightweight DB updates; Resend has documented retry schedule (5s, 5min, 30min, 2hr, 5hr, 10hr) limiting volume |
| T-03-18 | Information Disclosure | service_role key | mitigate | key only accessed via Deno.env.get() inside Edge Function; never in response body; webhook response is always 'ok' or 'Invalid signature' — no internal data leaked |
</threat_model>
