3. Copy the contents of supabase/migrations/011_csv_personalized.sql:

-- Migration 011: CSV Personalized Campaigns (Phase 9)
ALTER TABLE public.campaigns
  DROP CONSTRAINT campaigns_campaign_type_check,
  ADD CONSTRAINT campaigns_campaign_type_check
    CHECK (campaign_type IN ('regular', 'ab_test', 'ab_variant', 'csv_personalized'));

ALTER TABLE public.campaign_recipients
  ADD COLUMN IF NOT EXISTS personalized_subject TEXT,
  ADD COLUMN IF NOT EXISTS personalized_body TEXT;

4. Run the SQL
5. Verify constraint: SELECT constraint_name FROM information_schema.table_constraints WHERE table_name = 'campaigns' AND constraint_type = 'CHECK';
6. Verify columns: SELECT column_name FROM information_schema.columns WHERE table_name = 'campaign_recipients' AND column_name IN ('personalized_subject', 'personalized_body');

IMPORTANT: If the constraint name is different (not campaigns_campaign_type_check), the DROP will fail. In that case run SELECT conname FROM pg_constraint WHERE conrelid = 'public.campaigns'::regclass AND contype = 'c'; to find the actual name and adjust the DROP statement.
