Follow the CampaignBuilderPage pattern for page structure (max-w-4xl container, section headings with uppercase tracking-wide labels, Button/Card/Input from ui components, useNavigate for redirects, useToast for feedback).

**Imports:**
- `useState` from react
- `useNavigate` from react-router-dom
- `UploadCloud, X, FileText` from lucide-react
- `parseCsvFile, truncateBody` from `../../lib/csvParser`
- `useCsvCampaign` from `../../hooks/campaigns/useCsvCampaign`
- `useToast` from `../../components/ui/Toast`
- `Button` from `../../components/ui/Button`
- `Card` from `../../components/ui/Card`
- `Spinner` from `../../components/ui/Spinner`
- `CsvRow` type from `../../types/database`

**State:**
- `csvRows: CsvRow[]` -- parsed rows (empty initially)
- `parseError: string | null` -- column validation or parse error
- `fileName: string | null` -- name of selected file
- `campaignName: string` -- user-entered campaign name
- `creating: boolean` -- loading state for create button
- `dragActive: boolean` -- drag-over visual state

**Layout structure:**

1. **Page header**: `<h1>` "CSV Personalized Campaign" with a "Back to campaigns" secondary button linking to `/campaigns`

2. **Section 1 -- Upload CSV**:
   - Dropzone area: a `<div>` styled as a dashed-border drop target (border-gray-600, bg-gray-900/50). On drag over, highlight with border-indigo-500.
   - Contains a hidden `<input type="file" accept=".csv">` triggered by clicking the dropzone.
   - Icon: `<UploadCloud>` centered, text "Drop your CSV file here or click to browse"
   - Below the icon, small text: "Required columns: first_name, last_name, email, subject, body"
   - On file select (via input onChange or onDrop):
     a. Call `parseCsvFile(file)`
     b. If result.error, set `parseError` and clear `csvRows`
     c. If success, set `csvRows` to result.rows, set `fileName`, clear `parseError`
   - Handle drag events: `onDragOver` (preventDefault, setDragActive(true)), `onDragLeave` (setDragActive(false)), `onDrop` (preventDefault, setDragActive(false), extract file from e.dataTransfer.files[0])

3. **Error display**: If `parseError`, show a red alert box (bg-red-900/50 border-red-700 text-red-300 rounded-lg p-4) with the error message and a dismiss X button.

4. **File info bar**: If `fileName` is set (file loaded successfully), show a bar with FileText icon, file name, row count (`{csvRows.length} rows`), and a "Remove" ghost button that clears all state back to initial.

5. **Section 2 -- Preview** (only shown when csvRows.length > 0):
   - Section heading "Preview (first 5 rows)"
   - A Card with a table inside. Table columns: #, First Name, Last Name, Email, Subject, Body Preview
   - Show `csvRows.slice(0, 5)` as rows
   - Body column uses `truncateBody(row.body, 100)` -- plain text, max 100 chars
   - Style: standard dark table matching CampaignsPage (bg-gray-900 header, border-gray-800 rows, text-sm)

6. **Section 3 -- Campaign Details** (only shown when csvRows.length > 0):
   - Campaign name input using the Input component: label "Campaign name", placeholder "e.g. April newsletter personalized"
   - If campaignName is empty on submit, show validation error

7. **Bottom action bar** (only shown when csvRows.length > 0):
   - "Create campaign" primary Button, loading state `creating`
   - On click:
     a. Validate campaignName is not empty
     b. Set `creating = true`
     c. Call `createCsvCampaign({ name: campaignName, rows: csvRows })`
     d. If error, show toast with error message
     e. If success, show toast "Campaign created with {recipientCount} recipients" and `navigate(`/campaigns/${data.campaignId}/csv-review`)`
     f. Set `creating = false`

**Styling notes:**
- 2-space indentation, single quotes, PascalCase component name
- Dark theme: bg-gray-950/900 backgrounds, gray-100 text, indigo-500 accents
- Responsive: single column layout within max-w-4xl
- Match existing page patterns (CampaignBuilderPage sections structure)
