## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing (gap-2), table cell internal gaps |
| md | 16px | Default element spacing (gap-4), form field gaps |
| lg | 24px | Section padding (gap-6), Card padding md = p-6 |
| xl | 32px | Layout gaps (gap-8) |
| 2xl | 48px | Empty state vertical padding (py-16 = 64px, see exception) |
| 3xl | 64px | Page-level spacing |

Exceptions: Empty state vertical padding uses `py-16` (64px) — verified from CampaignsPage empty state pattern. Tab nav bottom border uses `-mb-px` overlap (not a spacing token — structural trick to merge tab border with container border).

Source: Verified from `CampaignsPage.tsx`, `ProfilePage.tsx`, `Card.tsx`.

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label | 14px | 600 (semibold) | 1.5 | `text-sm font-semibold` |
| Heading (card) | 14px | 600 (semibold) | 1.2 | `text-sm font-semibold` |
| Page title | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` |

Notes:
- Body text in tables, descriptions, and secondary info: `text-sm text-gray-300` or `text-sm text-gray-400`
- Table row primary value (name, subject): `text-sm font-semibold text-gray-100`
- Muted/tertiary info (date, subtext): `text-xs text-gray-500` or `text-sm text-gray-400`
- Monospace (workspace ID): `font-mono text-xs text-gray-500`
- Table headers: `text-xs font-semibold text-gray-400 uppercase tracking-wide`
- Two weights only: 400 (regular) and 600 (semibold). `font-medium` (500) is NOT used in this phase.

Source: Verified from `ProfilePage.tsx`, `CampaignsPage.tsx`, `AppLayout.tsx`.

## Page Layouts

### TemplatesPage (`/templates`)

Layout: `flex flex-col gap-6` — matches CampaignsPage exactly.

**Focal point:** The table body (`<tbody>`) is the primary focal zone. The first column (template name, `text-sm font-semibold text-gray-100`) draws the eye down the list. All other columns (subject, date, actions) are subordinate. No competing focal elements on this page — the header row is intentionally de-emphasized (`text-xs font-semibold text-gray-400 uppercase tracking-wide`).

Structure:
1. Page header row: `flex items-center justify-between`
   - Left: `<h1 className="text-xl font-semibold text-gray-100">Templates</h1>`
   - Right: No top-level CTA (templates are saved FROM CampaignsPage / CampaignBuilderPage, not created from scratch here)
2. Card container: `<Card padding="sm" className="overflow-hidden p-0">`
   - Loading state: centered `<Spinner size="lg" />` at `py-16`
   - Empty state: centered flex column at `py-16` (see Copywriting Contract)
   - Data state: `overflow-x-auto` table

Table columns:
| Column | Width | Alignment | Style |
|--------|-------|-----------|-------|
| Template name | auto | left | `text-sm font-semibold text-gray-100` |
| Subject line | auto | left | `text-sm text-gray-300` |
| Date saved | auto | left | `text-sm text-gray-400` |
| Actions | `w-12` | center | Ghost `MoreHorizontal` button with `aria-label="Template actions"` |

Row actions dropdown (same pattern as CampaignsPage):
- "Use template" — `text-sm text-gray-200 hover:bg-gray-700`
- "Delete" — `text-sm text-red-400 hover:bg-gray-700`

Source: D-01, D-02, D-03, D-04 from CONTEXT.md. Pattern: `CampaignsPage.tsx` verified.

### SettingsPage (`/settings`)

Layout: `max-w-2xl flex flex-col gap-6` — matches ProfilePage container width.

**Focal point:** The primary input card for the active tab is the focal zone. Specifically: on the Profile tab, the "Personal Information" card's first input (full name) is the primary focal element. On the Workspace tab, the "Sending Defaults" card's first input (default sender name) is the primary focal element. On the Integrations tab, the "Resend Integration" card's API key input is the primary focal element. Tab navigation is a secondary element and must not visually compete with the active card.

Structure:
1. Tab navigation strip: `flex gap-0 border-b border-gray-800 mb-6`
   - Three tabs: Profile, Workspace, Integrations
   - Active tab: `border-b-2 border-indigo-500 text-indigo-400 font-semibold text-sm px-4 py-2.5 -mb-px`
   - Inactive tab: `border-b-2 border-transparent text-gray-400 hover:text-gray-200 font-semibold text-sm px-4 py-2.5 -mb-px`
2. Tab content area: conditionally rendered based on `?tab=` param

**Profile tab** — existing ProfilePage content migrated verbatim:
- Avatar + name/email header row
- Card: "Personal Information" — full name, company name, timezone select
- Card: "Account" — email (read-only), Workspace ID (read-only, `font-mono`)
- Save button: right-aligned `Button variant="primary"` with `loading` state

**Workspace tab** — new content:
- Card: "Sending Defaults"
  - Input: "Default sender name" (type="text")
  - Input: "Default sender email" (type="email")
  - Right-aligned save button

**Integrations tab** — new content:
- Card: "Resend Integration"
  - Input: "API key" (type="password", always masked — see D-11)
    - When key exists: `placeholder="••••••••••••••••"`, value empty, `autoComplete="off"`
    - When no key: `placeholder="sk_live_..."`, `autoComplete="off"`
  - Read-only field: "Sending domain"
    - Label: `text-sm font-semibold text-gray-300`
    - Value: `text-sm text-gray-400` with `Badge variant="success"` showing "Active"
    - Static text: "resend.dev shared domain" — no input element
  - Textarea: "Unsubscribe footer text"
    - Matches Input styling: `bg-gray-800 border border-gray-700 rounded-lg px-4 py-2.5 text-gray-100 text-sm`
    - Default content from DB: `To unsubscribe from future emails, click here: {{unsubscribe_url}}`
  - Right-aligned save button

Source: D-08 through D-12 from CONTEXT.md.

## Interaction Contracts

### Tab switching (SettingsPage)

- State stored in `?tab=profile|workspace|integrations` via `useSearchParams`
- Default (no param): Profile tab active
- Tab click: `setSearchParams(tab === 'profile' ? {} : { tab })`
- No page reload — SPA transition only
- Each tab has its own independent save button and save handler
- Switching tabs does NOT trigger unsaved-changes warnings (no `beforeunload` guard — consistent with existing app patterns)

### "Save as template" flow (from CampaignsPage row action)

1. User clicks "..." menu on a campaign row
2. Dropdown shows new item: "Save as template" above "Edit" item
3. Click sets `saveAsTemplateTarget` state to the campaign object; closes dropdown
4. `SaveAsTemplateModal` renders with `campaign.name` pre-filled
5. User edits name (optional), clicks "Save template"
6. `createTemplate()` is called; button shows loading spinner
7. On success: modal closes, Toast "Template saved." (success)
8. On error: Toast with error message (error); modal stays open

### "Save as template" flow (from CampaignBuilderPage)

1. Button in page header/action area: "Save as template" (`Button variant="secondary" size="md"`)
2. Opens same `SaveAsTemplateModal` with `name` state value (fallback: "Untitled campaign")
3. Same save flow as above

### "Use template" action

1. User clicks "..." menu on a template row
2. Dropdown item: "Use template"
3. Click triggers `navigate('/campaigns/new?from_template=' + template.id)`
4. No confirmation modal — direct navigation (D-07)
5. CampaignBuilderPage reads `?from_template` param on mount and pre-fills fields

### Template delete

1. User clicks "..." menu on a template row
2. Dropdown item: "Delete" (`text-red-400`)
3. Click triggers `window.confirm("Delete '{name}'? This cannot be undone.")`
4. If confirmed: `deleteTemplate(id)` called; on success Toast "Template deleted." (success)
5. Table updates immediately via `fetchTemplates()` refetch

Source: window.confirm pattern — verified from `CampaignsPage.tsx` line 75.

### Settings save

- Each tab has its own `handleSave` function and its own `Button loading` state
- On success: Toast "Saved successfully." (success) + `refreshProfile()` called
- On error: Toast with error message (error)
- API key field: only included in Supabase UPDATE if `apiKeyDirty === true`

### Route redirect

- `/settings/profile` redirects to `/settings` via React Router `<Navigate to="/settings" replace />`
- Sidebar navItems: Settings entry updated from `/settings/profile` → `/settings`
- AppLayout pageTitles: add `/settings` → `'Settings'`; existing `/settings/profile` entry can remain for safety
