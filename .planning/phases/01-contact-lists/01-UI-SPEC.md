| sm | 8px | Compact stacked spacing (`gap-2`, nav list `gap-0.5` rows) |

| lg | 24px | Section padding (`p-6`, Card padding-md) |
| xl | 32px | Layout gaps (`p-8`, Card padding-lg) |
| 2xl | 48px | Major section breaks (between filter row and table) |

- Touch targets for icon-only buttons: minimum 32px (Tailwind `w-8 h-8`) — observed in Sidebar collapse button
- Drawer width: 384px (`w-96`) — fixed, not a spacing token
- Main content area padding: `p-4 lg:p-6` (16px mobile, 24px desktop) — from AppLayout

**60/30/10 assignment:**
- 60% — `gray-950`: page background, the dominant surface the user sees most
- 30% — `gray-900`: cards, sidebar, drawer, modal surfaces layered on top of background
- 10% — `indigo-600` / `indigo-400`: primary interactive elements listed below

**Accent (`indigo`) reserved for:**
1. Primary Button (Import Contacts, Create List, Save changes)
2. Focus ring on all interactive form elements (`focus:ring-indigo-500`)
3. Active nav item background (`indigo-600/20`) and text (`indigo-400`)
4. Sidebar logo mark (`bg-indigo-600`)
5. Wizard step progress indicator (active step dot)

**Accent is NOT used for:** hover states on ghost buttons, table row highlights, filter chips, drawer section headings, list color chips, tag badges.

**Destructive (`red-700`) reserved for:**
- Delete Contact button (inside drawer)
- Delete List button/action (in Lists tab)

**Semantic contact status colors (via Badge component):**
- `active` status → `Badge variant="success"` (green-400 text, green-900/50 bg)
- `unsubscribed` status → `Badge variant="warning"` (yellow-400 text, yellow-900/50 bg)
- `bounced` status → `Badge variant="danger"` (red-400 text, red-900/50 bg)
- `complained` status → `Badge variant="warning"` (same as unsubscribed)

## Interaction Contracts

### Tab Navigation (D-01 to D-05)

- Two tabs: "All Contacts" and "Lists"
- Tab bar renders directly below the page header, above the content area
- Active tab: `text-indigo-400` text + `border-b-2 border-indigo-500` underline (2px, flush to content area top edge)
- Inactive tab: `text-gray-400` text, `hover:text-gray-200` on hover, no underline
- Tab background: none — tabs float over `gray-950` background
- URL state: `/contacts` = All Contacts tab active; `/contacts?list=<id>` = Lists tab active with that list filtered
- Tab switch does NOT cause a full page reload — `setSearchParams({})` to clear

### Contacts Table (D-02, CONT-06 to CONT-09)

- Layout: full-width table inside a `Card` (padding sm — `p-4`)
- Columns: Checkbox | Name (first + last) | Email | Status (Badge) | Tags (Badge pills, max 3 visible + "+N more") | Added (relative date) | Actions (...)
- Row height: 48px (Tailwind `h-12` or `py-3`)
- Row hover: `hover:bg-gray-800/50` — subtle highlight, no border change
- Clicking a row (anywhere except checkbox or actions) opens the ContactDrawer
- Pagination: 50 rows per page, Previous/Next controls below table, page indicator "Page 1 of N"
- Pagination controls: `Button variant="secondary" size="sm"` for Previous/Next
- Empty table cell text: `text-gray-500 text-sm`
- Table header: `text-xs font-medium text-gray-400 uppercase tracking-wide bg-gray-900 border-b border-gray-800`

### Filter Bar (CONT-06 to CONT-09)

- Layout: horizontal row, `flex gap-3 items-center`, above the table
- Search input: `Input` with `Search` Lucide icon, placeholder "Search contacts..."
- Status filter: native `<select>` styled with `bg-gray-800 border border-gray-700 text-gray-100 text-sm rounded-lg px-3 py-2`; options: All Status, Active, Unsubscribed, Bounced, Complained
- Tag filter: `Input` (plain text), placeholder "Filter by tag..."
- Custom field filter: Two `Input` fields side-by-side — "Field name" + "Field value", placeholder per field
- Apply filters button: none — filters are reactive (300ms debounce via `useEffect`)
- Clear filters: "Clear filters" `Button variant="ghost" size="sm"` appears only when any filter is active

### Contact Drawer (D-06 to D-09)

- Position: `fixed right-0 top-0 h-full` — full viewport height, right edge
- Width: `w-96` (384px) on desktop; full-width on mobile (`w-full`)
- Background: `bg-gray-900 border-l border-gray-800 shadow-xl z-50`
- Backdrop: `fixed inset-0 bg-black/40 z-40` — clicking backdrop closes drawer
- Open animation: slide in from right (`translate-x-full` → `translate-x-0`, `transition-transform duration-200`)
- Header: `flex items-center justify-between p-4 border-b border-gray-800` — contact name (or "New Contact") left, X button right
- Close button: `Button variant="ghost" size="sm"` with `X` Lucide icon
- Body: `p-4 flex flex-col gap-4 overflow-y-auto`
- Sections within drawer: Contact Fields | List Membership | Danger Zone
- Section headings: `text-xs font-medium text-gray-400 uppercase tracking-wide mb-2`
- Edit inline: fields are read-only by default; "Edit" button toggles to editable `Input` components; "Save" / "Cancel" replace "Edit"
- Delete action: in "Danger Zone" section at drawer bottom; `Button variant="danger" size="sm"` labeled "Delete Contact"

### CSV Import Wizard (D-10 to D-17)

- Trigger: "Import Contacts" `Button variant="primary"` on contacts page header
- Modal: `fixed inset-0 bg-black/60 z-50 flex items-center justify-center` backdrop; modal panel `bg-gray-900 border border-gray-800 rounded-xl shadow-2xl w-full max-w-2xl mx-4`
- Step progress indicator: 4 numbered steps at top of modal; active step circle `bg-indigo-600 text-white`; completed step `bg-indigo-600/20 text-indigo-400 with checkmark`; upcoming step `bg-gray-800 text-gray-400`
- Step labels: "Upload" | "Map Columns" | "Preview" | "Confirm"
- File upload zone (Step 1): dashed border `border-2 border-dashed border-gray-700 rounded-lg p-8 text-center`; hover state `border-indigo-500 bg-indigo-600/5`; Upload icon (`UploadCloud` Lucide) above text "Drop a CSV file or click to browse"
- Column mapping (Step 2): table layout — detected CSV header left column, dropdown right column mapping to `email` | `first_name` | `last_name` | `company` | `(skip column)`; email mapping required (red asterisk `text-red-400` + validation message if not mapped)
- Duplicate strategy (Step 2, below column map): radio group — "Skip duplicate emails (recommended)" (default selected) | "Update existing contacts with new data"; radio label `text-sm text-gray-200`; description below `text-xs text-gray-400`
- Tag assignment (Step 2, below duplicate strategy): `Input` field "Apply tags to all imported contacts (optional)", placeholder "e.g. newsletter, 2026-import"; comma-separated; parsed into tag array
- Preview (Step 3): table showing first 5 CSV rows with mapped column values; summary block above: "N contacts to import, N duplicates detected (will be skipped/updated), N rows with missing email (skipped)"; text `text-sm text-gray-200`
- Confirm (Step 4): progress bar during import (`bg-indigo-600` fill, `bg-gray-800` track, `rounded-full h-2`); success state shows final counts; error state shows error summary
- Navigation buttons: "Back" `Button variant="secondary" size="md"` left, "Next" / "Start Import" `Button variant="primary" size="md"` right; "Next" disabled if required mapping not complete

### Lists Grid (D-03, LIST-01 to LIST-05)

- Layout: `grid grid-cols-1 sm:grid-cols-2 xl:grid-cols-3 gap-4`
- List card: `Card padding="sm"` with left color bar (4px wide, `rounded-l-lg`, color from preset palette)
- Card content: list name (`text-sm font-semibold text-gray-100`) + description (`text-xs text-gray-400 mt-0.5`) + contact count (`text-xl font-bold text-gray-100 mt-3`) + "contacts" label (`text-xs text-gray-500`)
- Card actions (top-right): `...` menu button (`Button variant="ghost" size="sm"`) revealing "Rename" and "Delete list" options
- Clicking a list card navigates to `/contacts?list=<id>` — shows filtered contacts view
- "New List" card: dashed border card `border-2 border-dashed border-gray-700 rounded-lg p-4 flex items-center justify-center gap-2 text-gray-400 hover:border-indigo-500 hover:text-indigo-400 cursor-pointer transition-colors`; Plus icon + "New List" label

### Create List Modal (LIST-01)

- Modal: `bg-gray-900 border border-gray-800 rounded-xl shadow-2xl w-full max-w-md mx-4`
- Fields: "List name" `Input` (required) | "Description" `Input` (optional) | "Color" ColorPicker
- ColorPicker: 8 color chips in a flex row; chip size `w-7 h-7 rounded-full border-2`; selected chip has `border-white`; unselected `border-transparent`; colors: `#6366f1` (indigo), `#22c55e` (green), `#f59e0b` (amber), `#ef4444` (red), `#06b6d4` (cyan), `#ec4899` (pink), `#8b5cf6` (violet), `#64748b` (slate)
- Actions: "Cancel" `Button variant="ghost"` + "Create List" `Button variant="primary" loading={creating}`
- Close: X button top-right, or clicking backdrop

### Import History Modal (D-18, D-19, CONT-04)

- Triggered by: "View import history" `Button variant="ghost" size="sm"` on contacts page header area
- Modal: `bg-gray-900 border border-gray-800 rounded-xl shadow-2xl w-full max-w-lg mx-4`
- Content: scrollable list of past import records; each row shows: date (relative, e.g. "2 days ago"), total rows, imported, skipped, errors counts; columns use `text-sm text-gray-200` for numbers, `text-xs text-gray-500` for labels
- Error detail: if row has errors > 0, expandable chevron reveals error details list (`text-xs text-red-400`)
- Empty state: "No imports yet" centered with Upload icon in `text-gray-600`

### Filtered List View (D-04, D-05, LIST-04)

- When `/contacts?list=<id>` is active: Lists tab is highlighted; above the contacts table a breadcrumb strip shows: colored dot (list color) + list name + "All Contacts" link to clear filter
- Breadcrumb strip: `flex items-center gap-2 mb-4 text-sm`; list name `text-gray-100 font-medium`; "All Contacts" link `text-indigo-400 hover:text-indigo-300`; separator `text-gray-600 /`
- The contacts table renders normally but scoped to list members only

## Accessibility

- All interactive elements must have visible focus rings: `focus:outline-none focus:ring-2 focus:ring-indigo-500` (already on Input; apply to all buttons and selects)
- Modal and drawer trap focus when open (`tabIndex` management or `inert` on background)
- Color chip in ColorPicker must also have `aria-label="{color-name}"` and `role="radio"` with `aria-checked`
- Contact status badges must not rely on color alone — Badge text always included (`active`, `unsubscribed`, `bounced`)
- Wizard step progress must have `aria-label="Step N of 4: {step name}"` on active step
- Delete confirmation must be announced via `aria-live="polite"` region
